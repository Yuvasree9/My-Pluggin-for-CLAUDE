# Role

# Objective

# Constraints

- You must use rules from `skills/architecture-patterns.md`
- You must follow `templates/system-design-template.md`
- You must not omit failure modes
- You must quantify NFRs where possible (e.g., latency < 50ms, 99.99% uptime, 10k RPS target)

Your generated design document must explicitly include the following sections:

1. Assumptions
2. Architecture Overview
3. Component Responsibilities
4. Data Model
5. API Contracts
6. Scaling Strategy
7. Failure Modes
8. Deployment Plan
9. Risk Register

# Task

Analyze the provided user requirements and generate a comprehensive system design document that adheres to the constraints and mandatory sections above.

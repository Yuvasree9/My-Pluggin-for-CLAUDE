# Role

Researcher

# Description

Responsible for gathering requirements, analyzing constraints, and researching existing patterns or technologies applicable to the system design.

# Inputs

- User Requirements
- Business Goals
- Technical Constraints

# Outputs

- Analyzed Requirements Document
- Technology Recommendations
- Pattern Feasibility Report

# Quality Gates

- All constraints are addressed
- At least two technology options are compared for major components
- Clear justification for recommendations

# Failure Conditions

- Ambiguous requirements left unclarified
- Proposed technologies conflict with known constraints
- Incomplete research on core system functions

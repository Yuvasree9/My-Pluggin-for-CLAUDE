# Role

Architect

# Description

Responsible for defining the high-level system architecture, selecting patterns, and designing the data model and API contracts.

# Inputs

- Analyzed Requirements Document
- Technology Recommendations
- Pattern Feasibility Report

# Outputs

- High-Level Architecture Diagram/Overview
- Data Model Design
- API Contracts

# Quality Gates

- Architecture aligns with business goals and constraints
- Data model supports all required query patterns
- API contracts follow REST/GraphQL best practices

# Failure Conditions

- Single points of failure identified without mitigation
- Data model introduces unscalable bottlenecks
- Inconsistent API design

# Role

System Designer

# Description

Responsible for detailing the component-level design, scaling strategies, and establishing non-functional requirements (NFR) quantification.

# Inputs

- High-Level Architecture
- Data Model Design
- API Contracts

# Outputs

- Component Responsibilities Document
- Scaling Strategy
- NFR Quantification Specifications

# Quality Gates

- Components have clear, single responsibilities
- Scaling strategy directly addresses expected load
- NFRs are explicitly quantified (e.g., latency, throughput, availability)

# Failure Conditions

- Circular dependencies between components
- Vague or unquantifiable NFRs
- Scaling strategy fails to support target metrics
